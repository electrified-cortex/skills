# Skill Index

Inline toolkit overview (read, do not dispatch). Three sub-skills handle the work: `skill-index-building` (produce artifacts), `skill-index-auditing` (validate cascade, write stamp on PASS), `skill-index-crawling` (consume cascade to locate a skill).

Artifact roles per directory:
- `skill.index` — raw index, authoritative, mechanically generated by builder
- `skill.index.md` — metadata overlay, LLM-authored, generated by builder
- `skill.index.sha256` — SHA-256 of raw bytes, written by auditor on PASS only

Builder produces `skill.index` + `skill.index.md`. Auditor validates then writes `skill.index.sha256` on PASS. Absence of stamp after build = "unaudited since last build," not "needs rebuild."

Two-step sequence: builder runs → auditor runs → auditor writes stamp on PASS. Builder never writes stamp.

Core rules:
- Raw index authoritative over overlay
- Cascade graph must be acyclic; auditor enforces
- Dot-folders skipped by default; bare-name allow-list overrides
- Symlinks not followed by default
- Index files must NOT use `.md` extension
- Toolkit does not open, execute, or audit skill contents
- Builder never consults stored stamp for change detection — compares recomputed hash against stored `skill.index` bytes

Combo node: directory that is simultaneously a leaf skill and a sub-node parent; emits self entry (`.`) in own index, marked combo in parent index; traversal not suppressed.

All three sub-skills are dispatch skills — invoke each via Dispatch agent (zero context) using `instructions.txt` in the sub-skill directory.

Related: `skill-index-building`, `skill-index-auditing`, `skill-index-crawling`, `skill-writing`
