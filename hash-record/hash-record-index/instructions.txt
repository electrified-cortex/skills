Params:
`repo_root` (required): abs path to repo root containing `.hash-record/`.
`--max-age-hours <N>` (optional): hours before `.meta.yaml` is stale. Default: 24.

Procedure:
Run every step with named tool. Don't summarize or plan.

0. Validate: `repo_root` contains `..` or shell metacharacters → output `ERROR: invalid repo_root`, stop.

1. Verify store: Bash `test -d "<repo_root>/.hash-record"`. Absent → output `CLEAN`, stop.

2. Collect hash dirs: Bash:
   ```bash
   find "<repo_root>/.hash-record" -mindepth 2 -maxdepth 2 -type d \
     ! -name '.prune' ! -name '.*'
   ```
   Each path: `<repo_root>/.hash-record/<shard>/<full-hash>`. None → output `CLEAN`, stop.

3. Max-age threshold: default `--max-age-hours` to 24. Cutoff = UTC now − <N> hours.

4. Per hash dir:
   a. Bash `test -f <hash_dir>/.meta.yaml`.
   b. Exists → Read, parse `last_seen`. More recent than cutoff → skip.
   c. Absent or `last_seen` ≤ cutoff → index.

5. Index hash dir:
   a. Bash:
      ```bash
      find "<hash_dir>" -type f -name "*.md" ! -name ".meta.yaml"
      ```
   b. Read each leaf. Extract `file_path` from YAML frontmatter. Absent/invalid frontmatter → skip silently.
   c. Collect non-empty `file_path` values. Deduplicate. Sort lexically.
   d. `.meta.yaml` exists → Read `file_paths`, union with collected, deduplicate, sort.
   e. `file_paths` empty → skip; don't create empty meta.
   f. Bash: `date -u +"%Y-%m-%dT%H:%M:%SZ"`
   g. Write `<hash_dir>/.meta.yaml` (Write tool), exact content:
      ```yaml
      file_paths:
        - <path-1>
        - <path-2>
      last_seen: <ISO-8601-timestamp>
      ```

6. Count hash dirs with `.meta.yaml` written.

7. Output: count=0 → `CLEAN`; else → `indexed: <count>`

Output (one line):
`CLEAN` — no dirs needed indexing (all recent or store absent).
`indexed: <count>` — <count> meta files written.
`ERROR: <reason>` — pre-exec failure (e.g., `repo_root` not found, path-traversal rejected).

Rules:
Never modify leaf `.md` records; read-only.
Never delete — overwrite stale `.meta.yaml` in-place; don't delete+recreate.
Write ONLY to `.meta.yaml` at hash-dir level.
Sole writer of `.meta.yaml`; no other skill writes meta.
Merge existing `file_paths` — never lose indexed paths.
Skip leaves with missing/malformed frontmatter silently; never raise on bad records.
Don't follow symlinks in `.hash-record/`.
Reject `repo_root` with `..` or shell metacharacters.
Don't write meta for hash dirs with no leaf records with valid `file_path`.
