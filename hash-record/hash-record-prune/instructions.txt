Params:
`repo_root`: repo root w/ `.hash-record/`.
`--dry-run`: list orphans; no delete (default: delete).
`--limit <N>`: max hash dirs deleted/invocation.

Procedure (each step named tool):

1. Verify: Bash `test -d "<repo_root>/.hash-record"` → absent = `CLEAN`, stop.

2. Collect: Bash `find "<repo_root>/.hash-record" -mindepth 2 -maxdepth 2 -type d ! -name '.prune' ! -name '.*' ! -path "<repo_root>/.hash-record/.*/*"`
   Extract basename (full-hash). None = `CLEAN`, stop.

3. Validity per hash dir (choose by manifest presence):

   First: enumerate all active worktree roots.
   `git -C "<repo_root>" worktree list --porcelain` — collect each `worktree <path>` line.

   Manifest present (`<full-hash>/manifest.yaml` exists) — re-derive manifest hash:
   a. Read `file_paths` list from manifest YAML.
   b. For each path: search all worktree roots in order; use first root where file exists.
      If no root has the file → ORPHANED.
   c. `git hash-object <abs-path>` each; collect (repo-relative-path, blob-hash) pairs.
   d. Sort pairs lexically by path.
   e. Build manifest text: `<path> <blob-hash>\n` per line (final line also ends `\n`).
   f. Write to temp file; `git hash-object <tempfile>`. Result == dir name → VALID; else ORPHANED.

   No manifest — blob-hash fallback (build once; share across all non-manifest dirs):
   For each active worktree root:
     `git -C "<wt-root>" ls-files --cached --others --exclude-standard`
     Filter out paths starting with `.worktrees/`. `git hash-object` each remaining file; accumulate into union set.
   Hash in union set → VALID; else ORPHANED.

4. Orphan: all hash dirs marked ORPHANED in step 3.

5. `--dry-run` → output `dry-run: <count>`, stop.

6. Delete orphans (≤ `--limit`): Bash `rm -rf "<repo_root>/.hash-record/<shard>/<full-hash>"`
   No symlinks; no paths outside `.hash-record/`; stop at limit.

7. Prune empty shards: `rmdir "<repo_root>/.hash-record/<shard>" 2>/dev/null || true`.

8. Output: `CLEAN` | `dry-run: <count>` | `pruned: <count>`.

Exit codes:
`CLEAN`: no orphans.
`pruned: <count>`: deleted.
`dry-run: <count>`: found; not deleted.
`ERROR: <reason>`: failure (bad `repo_root`, path-traversal rejected).

Safety:
Never delete hash if in valid set. Never delete `.hash-record/` or dot-dirs. No symlinks. No out-of-bounds paths. Build set once; in-pass changes ignored. No age/model/op-kind pruning—orphan status only. Reject `repo_root` w/ `..` or shell chars. Partial deletion OK; re-run resumes.
Previously-deleted records cannot be recovered; pruning is forward-only.
