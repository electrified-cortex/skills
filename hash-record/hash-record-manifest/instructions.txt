Params:
`repo_root` (required): absolute path to repo root. Used to compute repo-relative paths.
`files` (required): list of file paths to include. May be absolute or repo-relative; skill normalizes all to repo-relative. Comma-separated or newline-separated both accepted.

Procedure:
Run every step with named tool. Don't summarize or plan.

1. Guard — validate inputs. Check `repo_root` is non-empty and `files` is non-empty after parsing.
   `repo_root` absent or empty → `ERROR: missing repo_root`, stop.
   `files` empty or missing → `ERROR: empty file set`, stop.
   Bash `test -d "<repo_root>"`. Dir doesn't exist → `ERROR: repo_root not found`, stop.

2. Per file — resolve and hash. For each path in `files`:
   a. Resolve to repo-relative form:
      Path starts with `repo_root`: strip `repo_root` prefix (and any leading `/` or `\`).
      Canonicalize path separators to `/` (replace all `\` with `/`).
      Do NOT follow symlinks.
   b. Compute absolute path: if original path was already absolute, use it; otherwise join `repo_root` and repo-relative path.
   c. Bash to get git blob hash:
      ```bash
      git hash-object "<absolute-path>"
      ```
      Output is 40-char hex string. Command fails (file missing, unreadable, or git error) → `ERROR: missing: <repo-relative-path>`, stop immediately.

3. Sort. Collect all (repo-relative-path, blob-hash) pairs. Sort lexically by repo-relative path (ascending, byte order).

4. Build manifest text. Construct string with one line per pair in sorted order:
   ```
   <repo-relative-path> <blob-hash>
   ```
   Each line ends with `\n`. Final line MUST also end with `\n` (no trailing newline omission).

5. Compute manifest hash. Bash to pipe manifest text through `git hash-object --stdin`:
   ```bash
   printf '%s' "<manifest-text>" | git hash-object --stdin
   ```
   Or write to temp file and pipe:
   ```bash
   printf '%s\n' <line1> <line2> ... | git hash-object --stdin
   ```
   Output is 40-char manifest hash.

6. Output. Print exactly one line to stdout:
   ```
   manifest: <40-char-hash>
   ```
   Stop. Don't write any file to disk. Don't invoke any other skill.

Output (one line):
`manifest: <40-char-hash>` — success.
`ERROR: <reason>` — failure (missing input, file not found, git error).

Rules:
Only `git` permitted as external tool. No other commands for hashing.
Do NOT include file metadata (mtime, mode, size). Content hash is the only signal.
Do NOT follow symlinks when resolving file paths.
Do NOT silently skip missing files. Missing input → `ERROR:`, stop immediately.
Do NOT deduplicate paths. Duplicate paths in input produce duplicate manifest lines; that is a caller bug.
Do NOT cache this skill's own output.
Do NOT write any record file. Manifest hash returns via stdout only.
Do NOT invoke any other hash-record sub-skill or consumer skill.
Path separators in manifest text MUST always be `/` regardless of host OS.
Sorting MUST be lexical by repo-relative path before building manifest text.
Trailing newline after final manifest line is MANDATORY.
