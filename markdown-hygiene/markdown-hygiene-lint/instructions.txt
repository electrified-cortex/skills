Inputs:
`<markdown_file_path>` (required) — absolute path to `.md` file.
`--lint-path <lint_path>` (required) — absolute path to write lint.md. Missing -> `ERROR: --lint-path required`, stop.
`--ignore <RULE>[,<RULE>...]` (optional) — rules to skip.

Hard prohibition: don't author scripts (`.ps1`, `.sh`, etc.), helper files, or any file other than `<lint_path>`. Target is read-only. Use Read/Bash/Grep only for inspection.

1. Run verify, whichever runtime has:
   Bash: `bash <skill-dir>/../verify.sh <markdown_file_path> [--ignore <RULE>[,<RULE>...]]`
   PS7: `pwsh <skill-dir>/../verify.ps1 <markdown_file_path> [-Ignore <RULE>[,<RULE>...]]`
   Include its output in findings. Rules covered by verify: MD009, MD010, MD012, MD041, MD047 — don't re-check these unless verify was skipped.
2. Scan `<markdown_file_path>` for remaining MD rules. Read file, apply rule-knowledge from step 3. Cross-check each finding against actual line; drop any not pointable to a verified line. Skip `--ignore` rules. Cover all rules including MD055/MD056/MD058/MD060.
   Adaptive MD041 suppression: if first non-blank line is `---` (YAML frontmatter), drop any MD041 finding.
3. Reference rules:
   MD001 — heading levels increment by one (H1 to H3 violates).
   MD003 — heading style consistent (atx `#` vs setext `===`/`---`).
   MD004 — list markers consistent (`-`, `*`, `+`).
   MD009 — trailing spaces.
   MD010 — hard tabs in non-code content.
   MD012 — multiple consecutive blank lines.
   MD022 — headings need blank before AND after (except start/end).
   MD023 — headings must not have leading whitespace.
   MD024 — duplicate heading text among siblings.
   MD025 — only one H1 per file.
   MD026 — headings must not end with `.`, `!`, `?`, `:`, `,`, `;`.
   MD029 — ordered list prefixes consistent.
   MD031 — fenced code blocks need blanks before AND after.
   MD032 — lists need blanks before AND after.
   MD033 — no inline HTML outside fenced blocks.
   MD034 — bare URLs (not in angle brackets, backticks, or links).
   MD040 — fenced code blocks need language identifier.
   MD041 — first non-blank line must be H1 (suppressed for frontmatter).
   MD047 — file must end with exactly one newline.
   MD055 — table pipe style consistent across rows.
   MD056 — all rows in table same number of cells.
   MD058 — tables need blanks before AND after.
   MD060 — table cell separators need space on each side of dash run.
4. Write `<lint_path>`:
   `mkdir -p $(dirname <lint_path>)` first.
   Frontmatter (`---` open/close):
     `file_path: <repo-relative path>` — via `git ls-files --full-name <markdown_file_path>` or strip `<repo_root>/`.
     `operation_kind: markdown-hygiene-lint`
     `result: clean` (no violations) or `result: fail` (violations present).
   Body:
     No violations: `# Result\n\nCLEAN`.
     Violations: `# Result\n\nFINDINGS\n\n- <list>`. Each entry two lines:
       `MD0XX line N: <description>`
       Indented `Fix: <imperative>` — complete, standalone, byte-precise.
5. Return `clean` (no violations) or `findings: <lint_path>` (violations present). On failure: `ERROR: <reason>`.
