Detect, fix markdownlint violations. Zero errors gates execution.

Params:
`file_path` (required): absolute path to .md file
`--filename <name>` (required): filename for record; lowercase-hyphenated; use VERBATIM — no inference, suffix, date, sub-version. Missing → `ERROR: --filename required`, stop.
`--fix` (optional): apply fixes after detect; without, detect-only — file untouched.
`--source X --target Y` (optional): read X, fix, write Y; X untouched, no git check; implies `--fix`.
`--ignore <RULE>[,<RULE>...]` (optional): skip rules — not scanned, flagged, fixed.
`--force` (optional): bypass cache HIT, re-execute despite cached record.
Adaptive MD041: first non-blank `---` (YAML) → auto-suppress MD041; no flag needed.

Modes:
Detect (default, no `--fix`): read-only, no git check; always runs; one detect record.
Fix (with `--fix`): `git status --porcelain -- <file>` → proceed if empty or `M ` (M+space); else `ERROR: target_dirty: stage, re-run; cached detect carries forward`, stop. Writes second record at post-fix hash.
Source→Target: `--source X --target Y` reads X, fixes, writes Y; X untouched; implies `--fix`.

Procedure:

Executor, not planner. Run each step's tool calls. Don't summarize, don't show "fixed version" inline. Don't stop early. Dispatch incomplete without record file + output path.

Two passes: detect (always), fix (only if `--fix` or `--source`/`--target`).

Pass 1 — Detect:

0. Guard: missing `--filename` → `ERROR: --filename required`, stop immediately.
1. Read target (Read tool). Guard: unreadable or non-.md → `ERROR: <reason>`, stop. Adaptive MD041: first non-blank `---` → add MD041 to suppressed set.
2. `git hash-object <file_path>` (Bash). Save 40-char as `<hash>`.
3. Cache check: `test -f <repo-root>/.hash-record/<hash[0:2]>/<hash>/markdown-hygiene/<filename>.md` (Bash) where `<filename>` = `--filename` value. Skip if `--force`.
   — Exists: read `result:` frontmatter — if `pass` output `CLEAN`; if `findings` output `findings: <path>` — stop (HIT).
   — Missing: MISS. Save `<repo-root>/.hash-record/<hash[0:2]>/<hash>/markdown-hygiene` as `<detect_cache_dir>` (no /). Continue.
4. Scan markdownlint violations. Prefer `markdownlint` CLI (see tooling.md); else use knowledge. Skip rules in `--ignore` or adaptive suppressed set. Cross-check findings vs actual lines — drop unreferenced.
   Per-rule anchors:
   — MD001: heading levels increment by one (H1→H3 violates).
   — MD003: heading style consistent (atx `#` vs setext `===`/`---`); flag headings differing from first.
   — MD004: list markers consistent (`-`, `*`, `+`); flag items using different marker than first in list.
   — MD009: trailing spaces; flag lines ending with 1+ spaces (except two-space line-break).
   — MD010: hard tabs; flag tabs in non-code content.
   — MD012: 2+ consecutive blanks; flag runs.
   — MD022: headings need blank before + after (except start/end).
   — MD023: headings no leading whitespace (start `#`).
   — MD024: duplicate heading text among siblings; flag identical text at same level/parent.
   — MD025: one H1 per file; flag every H1 after first.
   — MD026: headings no ending punctuation (. ! ? : , ;).
   — MD029: ordered list prefixes consistent (all `1.` or strictly incrementing).
   — MD031: code blocks need blank before + after.
   — MD032: lists need blank before + after.
   — MD033: no inline HTML; flag `<tag>`, `</tag>`, `<!-- comment -->` outside blocks.
   — MD034: bare URLs; flag `http://`, `https://` outside brackets, backticks, links.
   — MD040: code blocks need language ID (` ```python `, not bare ` ``` `).
   — MD041: first non-blank must be H1 (suppressed with frontmatter or `--ignore MD041`).
   — MD047: file ends exactly one newline.
   — MD055: table pipe style consistent across rows.
   — MD056: all table rows same cell count.
   — MD058: tables need blank before + after.
   — MD060: table separators spaced (`| --- |` not `|---|`).
5. Write detect record at `<detect_cache_dir>/<filename>.md`:
   — Filename: `<filename>.md` (value from `--filename`, VERBATIM; append nothing). Caller controls filename; executor uses supplied value as-is. Missing → stop with `ERROR: --filename required`.
     Correct: `.hash-record/<sh>/<hash>/markdown-hygiene/claude-haiku.md`
     Incorrect: appended date/qualifiers/inferred model.
   — `mkdir -p <detect_cache_dir>` (Bash).
   — Frontmatter: `hash: <hash>`, `file_path: <repo-relative>` (via `git ls-files --full-name` or strip `git rev-parse --show-toplevel`; e.g. `markdown-hygiene/instructions.uncompressed.md`), `operation_kind: markdown-hygiene`, `result: pass|findings`.
   — Body: Report Format. No violations: `# Result\n\nCLEAN`. Violations: `# Result\n\nFINDINGS\n\n- <list>`. Each entry: `MD0XX line N: <desc>` + indented `Fix: <imperative>`. Fix: standalone, literal.
   — No violations: output `CLEAN`, stop.
   — Violations, no `--fix`: output `findings: <detect_cache_dir>/<filename>.md`, stop.

Pass 2 — Fix (only when `--fix` + violations):

6. ALL violations unfixable (manual-only, e.g., MD040 no-language) → output `findings: <detect_record_path>`, stop (no second record).
7. Read detect record (Read tool). Extract `Fix:` lines from FINDINGS body. Apply literally via Edit/Write. No re-scan, no knowledge. File MUST contain all applied fixes.
8. Compute new hash: `git hash-object <fixed-file>`. Save as `<fix_hash>`. Set `<fix_cache_dir>` = `<repo-root>/.hash-record/<fix_hash[0:2]>/<fix_hash>/markdown-hygiene` (no /).
9. Write fix record at `<fix_cache_dir>/<filename>.md`:
   — `mkdir -p <fix_cache_dir>` (Bash).
   — Frontmatter: `hash: <fix_hash>`, `file_path: <repo-relative>`, `operation_kind: markdown-hygiene`, `result: pass|findings`.
   — Body: Report Format. All applied: `# Result\n\nFIXED\n\n- <list>`. Some remain: `# Result\n\nPARTIAL\n\n...`.
   — All fixed: output `CLEAN`, stop. Some remain: output `findings: <fix_cache_dir>/<filename>.md`, stop.

Dispatch return (one line):
Clean (no violations OR all fixed): `CLEAN`
Findings (remain — detect-only or PARTIAL): `findings: <absolute-path-to-record.md>`
Pre-write failure: `ERROR: <reason>`

Record body — minimum info not in frontmatter. Frontmatter: `file_path`, `result`, `hash`, `model`. Never dupe `file_path` in body. Body opens: `# Result`.

CLEAN:
```
# Result

CLEAN
```

FINDINGS (violations, no `--fix`) — each entry: `MD0XX line N: <desc>` + indented `Fix: <imperative>`:
```
# Result

FINDINGS

- MD022 line 7: blank missing before "Configuration"
  Fix: insert blank line before line 7
- MD047: file lacks trailing newline
  Fix: append single newline at end
```

FIXED (all resolved):
```
# Result

FIXED

- MD022: blanks around heading (1)
- MD047: trailing newline (1)
```

PARTIAL (some unfixable):
```
# Result

PARTIAL

Fixed:
- MD022 (1)

Remaining (manual):
- MD040 line 14: missing language ID
```

Verdict: CLEAN → pass; FIXED → pass; FINDINGS → findings; PARTIAL → findings; error → error.

Tables — two canonical modes, never mix per table:
Mode A (default): 1–3 dashes per cell, pipe-padded; widths flexible. e.g. `| - | --- | --- |`
Mode B (aligned): dashes match header width; use when readability matters.
Prohibitions: no unpadded pipes (`|---|---|` violates MD060; use `| --- | --- |`); no extra whitespace in headers (`|  Header  |` → `| Header |`); no inconsistent widths; no mixed Mode A + B.

Rules:
Fix every violation. Never suppress unless in `--ignore`.
Never change meaning — formatting only.
Never introduce violations while fixing.
Preserve code blocks, frontmatter, tech strings exactly.
Fix would change meaning → report unfixable.
One file per dispatch.
Use tools + knowledge; no external packages.
Intentional bad markdown: violation inside inline code or demonstrating incorrect syntax → preserve, report PARTIAL with line. Don't fix content demonstrating violation.
