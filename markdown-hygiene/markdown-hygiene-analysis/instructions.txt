Inputs:
`<markdown_file_path>` (required) ‚Äî absolute path to `.md` file.
`--lint-path <lint_path>` (required) ‚Äî absolute path to existing lint.md (read-only reference). Missing -> `ERROR: --lint-path required`, stop.
`--analysis-path <analysis_path>` (required) ‚Äî absolute path to write analysis.md. Missing -> `ERROR: --analysis-path required`, stop.
`--ignore <RULE>[,<RULE>...]` (optional) ‚Äî SA rules to skip.

Hard prohibition: don't author scripts (`.ps1`, `.sh`, etc.), helper files, or any file other than `<analysis_path>`. Target is read-only. Use Read/Bash/Grep only for inspection.

1. Read `<lint_path>` ‚Äî extract frontmatter `result` field (`clean` or `fail`) and FINDINGS list (count violations for index summary).
2. Semantic advisory scan: read `<markdown_file_path>`, evaluate SA001-SA038 below. Each entry uses `Note:` not `Fix:` ‚Äî host LLM decides whether to act. Skip `--ignore` rules. SA032 and SA038 require semantic reasoning ‚Äî flag only when clearly evident.
3. SA rules:
   SA001 [FAIL] ‚Äî 3+ consecutive ALL CAPS words not an acronym/known term.
   SA002 [FAIL] ‚Äî entire sentence in ALL CAPS.
   SA003 [WARN] ‚Äî same ALL CAPS word 5+ times in document (signal collapse).
   SA004 [WARN] ‚Äî >~15% non-code body text is bold (emphasis diluted).
   SA005 [WARN] ‚Äî bold spans 3+ consecutive sentences in same section.
   SA006 [FAIL] ‚Äî single-item list (use a sentence instead).
   SA007 [WARN] ‚Äî two-item list that reads naturally as `X and Y`.
   SA008 [WARN] ‚Äî list nesting 3+ levels deep.
   SA009 [WARN] ‚Äî list items are multi-sentence paragraphs (consider sections).
   SA010 [WARN] ‚Äî file path, shell command, or env var in plain prose without backticks.
   SA011 [FAIL] ‚Äî ALL CAPS and bold on same word/phrase (e.g. `**NEVER**`) ‚Äî redundant double-emphasis.
   SA012 [WARN] ‚Äî heading immediately followed by another heading, no content between.
   SA013 [WARN] ‚Äî heading introduces only a single sentence (use `**Label:**` instead).
   SA014 [SUGGEST] ‚Äî `never`/`must not`/`do not`/`always` unemphasized in instruction document.
   SA015 [WARN] ‚Äî document >400 words, zero headings.
   SA016 [WARN] ‚Äî heading >~60 chars.
   SA017 [WARN] ‚Äî heading phrased as question (ends `?`) outside FAQ context.
   SA018 [WARN] ‚Äî passive voice on directive sentence in instruction/spec document.
   SA019 [WARN] ‚Äî paragraph >8 sentences.
   SA020 [WARN] ‚Äî numbered list with no sequential dependency (use bullets).
   SA021 [WARN] ‚Äî link text is raw URL.
   SA022 [WARN] ‚Äî generic link text: "click here", "here", "this link", "read more".
   SA023 [FAIL] ‚Äî empty fenced code block.
   SA024 [WARN] ‚Äî backtick on plain prose word that is not code/path/command/literal.
   SA025 [WARN] ‚Äî strikethrough text in non-draft document.
   SA026 [WARN] ‚Äî `---`/`***` used as section divider where a heading should be used.
   SA027 [WARN] ‚Äî same word in every sibling heading within a section.
   SA028 [WARN] ‚Äî phrase of 5+ words or full sentence appears verbatim more than once.
   SA029 [WARN] ‚Äî positional reference: "see above", "see below", "as mentioned earlier", etc.
   SA030 [WARN] ‚Äî blockquote (`>`) used as callout/emphasis rather than quotation.
   SA031 [WARN] ‚Äî sibling headings mix Title Case and Sentence case.
   SA032 [WARN] ‚Äî same concept referred to by multiple distinct names (LLM-detected).
   SA033 [FAIL] ‚Äî negation stacking in directive (e.g. "do not fail to avoid") ‚Äî rewrite as positive.
   SA034 [WARN] ‚Äî vague qualifier on directive ("sometimes", "usually", "often", "generally") without stating exception condition.
   SA035 [WARN] ‚Äî action stated before gate condition ("Delete X if Y" ‚Äî prefer "If Y, delete X").
   SA036 [WARN] ‚Äî directive sentence has 3+ coordinating conjunctions ‚Äî split into one instruction per sentence.
   SA037 [WARN] ‚Äî list mixes command items and observation/description items without a distinguishing signal.
   SA038 [FAIL] ‚Äî contradictory instructions present in document (LLM-detected).
4. Determine overall result:
   lint_result = frontmatter `result` from `<lint_path>`.
   advisory_count = number of SA findings.
   overall = `fail` if lint_result == fail; `pass` if lint_result == clean AND advisory_count > 0; `clean` if both zero.
5. Write `<analysis_path>`:
   `mkdir -p $(dirname <analysis_path>)` first.
   Frontmatter: `file_path: <repo-relative>`; `operation_kind: markdown-hygiene-analysis`; `result: clean` (no advisories, lint clean), `result: pass` (advisories, lint clean), or `result: fail` (lint fail).
   Body:
     No advisories, lint clean: `# Result\n\nCLEAN`.
     Advisories: `# Result\n\n## Advisory\n\n- <list>`. Each entry two lines:
       `SA0XX [SEVERITY] line N: <description>` (omit line N for document-level)
       Indented `Note: <observation>` ‚Äî plain English, no imperative.
6. Return `clean` (no advisories, lint clean), `pass: <analysis_path>` (advisories, lint clean), or `findings: <analysis_path>` (lint fail). On failure: `ERROR: <reason>`.
