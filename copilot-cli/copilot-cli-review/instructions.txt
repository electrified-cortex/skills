Prerequisites:
```bash
copilot --version   # must resolve; fail-fast if not
```
Fails → return `Status: UNAVAILABLE`; surface stderr; stop. Do NOT attempt installation.

Invocation:
```bash
copilot -p "<prompt>" -s --allow-all-tools
```
Add `--model <model>` only when caller explicitly supplied model name. Omit otherwise; don't pin model.

Named models (e.g. `gpt-4o`, `gpt-4.5`) may be unavailable via `--model`. Use default unless caller requires named model and it's confirmed available.

Add `--output-format=json` when structured output needed AND installed version supports flag. Prefer JSON; fall back to markdown parsing otherwise.

Long-form equivalence: `--no-ask-user --prompt "<prompt>"` ≡ `-p "<prompt>" -s`. New invocations MUST use canonical short form.

Constrain `working_dir` to target repo — never `/`, `~`, or dirs containing secrets.

Prompt Template:
Embed all diff or file content inline. No file-input flag — serialize into prompt string.
```text
Review the following change set for correctness, security vulnerabilities, and code quality.
Return a structured findings list. Each finding must include:
  severity: blocker | major | minor | nit
  file: <path>
  line: <line number or range>
  description: <one sentence>
If there are no issues, respond with exactly: No findings.

<inline diff or file content>
```
Severity vocabulary fixed: `blocker / major / minor / nit`.

Output Shape:
```text
Status: CLEAN | FINDINGS | UNAVAILABLE | ERROR
Findings:
  - severity: blocker | major | minor | nit
    file: <path>
    line: <number or range>
    description: <one sentence>
Raw: <Copilot's full response, JSON or markdown>
```

| Status | Condition |
| --- | --- |
| `CLEAN` | Copilot responded with exactly "No findings." |
| `FINDINGS` | One or more findings returned |
| `UNAVAILABLE` | `copilot --version` failed before invocation |
| `ERROR` | Non-zero exit code or unparseable output |

Severity normalization: `critical` → `blocker`, `nitpick` → `nit`, others coerce to nearest or flag in description.

Error Handling:
| Condition | Action |
| --- | --- |
| `copilot --version` fails | `Status: UNAVAILABLE`; surface stderr; stop |
| Binary exits non-zero | `Status: ERROR`; surface stderr; stop |
| Output unparseable | `Status: ERROR`; include raw; stop |
| Model not available | `Status: ERROR`; don't retry with different model |

Rules:
Don't return raw Copilot output — parse to output shape first.
One repo per invocation. Don't fan out across dirs.
Don't bake in specific CLI version assumption — detect, then use.
Don't install, update, or upgrade Copilot CLI.
Don't advise on severity thresholds — caller's policy.
