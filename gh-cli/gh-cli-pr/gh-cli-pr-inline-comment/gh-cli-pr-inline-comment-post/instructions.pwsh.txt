# GH CLI PR Inline Comment — Post (pwsh 7+)

Prereq: `gh auth status 2>&1`

Step 1: Fetch commit SHA
Fetch fresh — stale SHAs → 422.
```powershell
$COMMIT_SHA = gh pr view $PR_NUMBER --repo "$OWNER/$REPO" --json headRefOid --jq '.headRefOid'
```

Step 2: Verify file in diff
```powershell
gh pr diff $PR_NUMBER --repo "$OWNER/$REPO" --name-only
```
FILE_PATH not listed → stop: no changes in this PR.

Step 3: Verify line in diff
Use bundled tool — don't parse diff manually.
SIDE not provided → default to `RIGHT` before invoking.
```powershell
$verifyOutput = pwsh verify-line-in-diff.ps1 -owner $OWNER -repo $REPO -pr_number $PR_NUMBER -file_path $FILE_PATH -line_number $LINE_NUMBER -side $SIDE
$VERIFY_EXIT = $LASTEXITCODE
```
Exit codes: 0 (IN_DIFF) → proceed; 1 (NOT_IN_DIFF) → outside hunk ranges, stop, surface valid ranges; 2 (FILE_NOT_IN_DIFF) → no changes, stop; 3 (USAGE_ERROR) → invalid args; 4 (API_ERROR) → `gh pr diff` failed, surface error.

WINDOWS: NEVER use leading `/` in `gh api` paths — use `repos/...` not `/repos/...`.

Step 4: Check existing comment (dedup)

4a — Positional match (primary):
```powershell
$existing = gh api --paginate "repos/$OWNER/$REPO/pulls/$PR_NUMBER/comments" `
  --jq ".[] | select(.path == \`"$FILE_PATH\`" and .side == \`"$SIDE\`" and (
    (\`"$SIDE\`" == \`"RIGHT\`" and .line == $LINE_NUMBER) or
    (\`"$SIDE\`" == \`"LEFT\`"  and .original_line == $LINE_NUMBER)
  )) | {id, body, author: .user.login}"
```

4b — Body-content fallback (eventual-consistency guard). PR comment index is eventually consistent; timed-out post + retry may arrive in 2–5s window before just-posted comment appears in positional query — keeps dedup idempotent on retry. Only run when 4a empty; match body (post-trim), same path + side.
```powershell
if ([string]::IsNullOrWhiteSpace($existing)) {
  $bodyTrimmed = $BODY.Trim()
  $allComments = gh api --paginate "repos/$OWNER/$REPO/pulls/$PR_NUMBER/comments" | ConvertFrom-Json
  $existing = $allComments |
    Where-Object { $_.path -eq $FILE_PATH -and $_.side -eq $SIDE -and ($_.body ?? '').Trim() -eq $bodyTrimmed } |
    Select-Object -First 1 |
    ForEach-Object { [pscustomobject]@{ id = $_.id; body = $_.body; author = $_.user.login } } |
    ConvertTo-Json -Compress
}
```

4a or 4b matched → return: `{ "status": "duplicate", "comment_id": <existing_id>, "comment_url": "https://github.com/{OWNER}/{REPO}/pull/{PR_NUMBER}#discussion_r<existing_id>", "message": "comment already exists at {FILE_PATH}:{LINE_NUMBER}" }`

Step 5: Post comment
BODY_FILE provided → use directly, no temp file:
```powershell
$bodyFile = $BODY_FILE
$createdTempFile = $false
```
BODY only → write to temp via VERBATIM single-quote here-string. NEVER assign BODY to double-quoted string — `@"..."@` expands `` `n ``→newline, strips `` `$ ``, consumes `` ` ``+char pairs. `@'...'@` passes content verbatim.
Embed BODY directly — don't use `$BODY` variable:
```powershell
$bodyFile = [System.IO.Path]::GetTempFileName()
$createdTempFile = $true
[System.IO.File]::WriteAllText($bodyFile, @'
<embed BODY verbatim here — replace this line with the exact body text, then leave the closing '@, on its own line>
'@, [System.Text.Encoding]::UTF8)
```
Multi-line BODY: each line in `@'...'@` as-is. BODY contains `'@` on own line → use different delimiter.

Invoke `post.ps1` (kebab-case flags, parsed manually — pass as `--flag`, not PS-style):
```powershell
$POST_URL = pwsh post.ps1 `
  --owner $OWNER `
  --repo $REPO `
  --pr $PR_NUMBER `
  --commit-sha $COMMIT_SHA `
  --file $FILE_PATH `
  --line $LINE_NUMBER `
  --side $SIDE `
  --body-file $bodyFile
$POST_EXIT = $LASTEXITCODE
if ($createdTempFile) { Remove-Item $bodyFile -Force -ErrorAction SilentlyContinue }
```

Step 6: Parse and return
Tool emits comment URL on stdout (exit 0) or nothing on error.
```powershell
switch ($POST_EXIT) {
  0 {
    $COMMENT_URL = $POST_URL.Trim()
    if ($COMMENT_URL -match '#discussion_r(\d+)') {
      $COMMENT_ID = $Matches[1]
    }
  }
  3 {
    Write-Output "{ `"status`": `"error`", `"comment_id`": null, `"comment_url`": null, `"message`": `"Line $LINE_NUMBER is not in the diff for $FILE_PATH`" }"
    exit 0
  }
  4 {
    Write-Output '{ "status": "error", "comment_id": null, "comment_url": null, "message": "gh api error — see stderr" }'
    exit 0
  }
  2 {
    Write-Output '{ "status": "error", "comment_id": null, "comment_url": null, "message": "internal error: bad args to post.ps1" }'
    exit 0
  }
  default {
    Write-Output ('{ "status": "error", "comment_id": null, "comment_url": null, "message": "post.ps1 exited with unexpected code ' + $POST_EXIT + '" }')
    exit 0
  }
}
```
