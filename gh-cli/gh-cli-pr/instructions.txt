GH CLI PRs — entry point for PR management via GitHub CLI. Handles PR inspection; routes write ops to sub-skills.

Use when: inspecting PRs (list, view, diff, check status) or want auto-routing for write ops. Know sub-skill → dispatch directly.

Inspection commands:
```bash
gh pr list --state open --author @me --json number,title,headRefName
gh pr view 123 --comments
gh pr diff 123
gh pr checks 123 --watch
gh pr status
```

Sub-skills:
| Sub-skill | Handles |
| --- | --- |
| gh-cli-prs-create/ | Open new PRs, convert drafts to ready |
| gh-cli-prs-review/ | Approve, request changes, dismiss reviews |
| gh-cli-prs-comments/ | Add, edit, delete PR comments |
| gh-cli-prs-merge/ | Merge, update branch, revert, close |

Notes:
Use `--repo owner/name` when not inside local clone of target repo.
`gh pr checks --watch` blocks until CI completes.
Scope: `gh pr` subcommands only. Excludes git ops, branch protection, CODEOWNERS.
