# GH CLI PR Create

Open pull requests using `gh pr create`. Covers creating standard PRs, draft PRs, and promoting drafts to ready.

## Before Creating — Check for Existing PR

Confirm no existing open PR for current branch:

```bash
gh pr list --head $(git branch --show-current)
```

If a PR already exists, edit or view it rather than creating a duplicate.

## Creating a Pull Request

Create with title, body, and base branch:

```bash
gh pr create --title "title" --body "body" --base main
```

Create with full metadata — reviewers, assignee, labels, start as draft:

```bash
gh pr create --title "title" --body-file .github/PULL_REQUEST_TEMPLATE.md \
  --reviewer user1,user2 --assignee @me --label enhancement --draft
```

## Linking to a Closing Issue

Include a closing keyword in PR body to automatically close linked issue on merge:

```bash
# In --body value or --body-file content:
Closes #123
```

## Promoting a Draft to Ready

When PR is ready for review:

```bash
gh pr ready 123
```

## Editing Metadata After Creation

Add reviewers, labels, or remove labels after PR is open:

```bash
gh pr edit 123 --add-reviewer user3 --add-label bug --remove-label wip
```

## Scope Boundaries

Covers `gh pr create`, `gh pr ready`, and `gh pr edit`. Doesn't cover branch creation or `git push` — branch must already exist on remote before running these commands. Reviewing and merging belong to their respective sub-skills.
