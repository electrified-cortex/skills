GH CLI PR Create — open PRs via `gh pr create`: standard, draft, promote draft to ready.

Before creating — confirm no existing open PR for current branch:
```bash
gh pr list --head $(git branch --show-current)
```
PR already exists → edit or view it; don't create duplicate.

Create with title, body, base:
```bash
gh pr create --title "title" --body "body" --base main
```

Create with full metadata:
```bash
gh pr create --title "title" --body-file .github/PULL_REQUEST_TEMPLATE.md \
  --reviewer user1,user2 --assignee @me --label enhancement --draft
```

Link to closing issue — include in body; auto-closes issue on merge:
```bash
# In --body value or --body-file content:
Closes #123
```

Promote draft to ready:
```bash
gh pr ready 123
```

Edit metadata after creation:
```bash
gh pr edit 123 --add-reviewer user3 --add-label bug --remove-label wip
```

Scope: `gh pr create`, `gh pr ready`, `gh pr edit` only. Excludes branch creation and `git push` — branch must already exist on remote. Reviewing and merging → respective sub-skills.
