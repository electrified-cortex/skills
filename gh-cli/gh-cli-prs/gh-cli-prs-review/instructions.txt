# GH CLI PR Review

Perform and dismiss pull request reviews using `gh pr review`.

## Approving a Pull Request

Approve with optional body message:

```bash
gh pr review 123 --approve --body "LGTM"
```

## Requesting Changes

Request changes. A body is required to explain what needs to be addressed:

```bash
gh pr review 123 --request-changes --body "Please address X before merging"
```

## Comment-Only Review (No Verdict)

Submit a review comment without approving or requesting changes:

```bash
gh pr review 123 --comment --body "Thoughts inline"
```

## Dismissing a Review

Dismiss a previously submitted review. `--review-id` is required — `gh` won't accept `--dismiss` without it:

```bash
gh pr review 123 --dismiss --review-id <review-id> --body "reason"
```

To obtain review ID:

```bash
gh pr view 123 --json reviews --jq '.reviews[].id'
```

## Adding Reviewers

To add reviewers at PR creation time, use `gh pr create --reviewer`. To add after creation:

```bash
gh pr edit 123 --add-reviewer user
```

These are covered by `gh-cli-prs-create`, not this skill.

## Resolving Review Threads

Resolving review threads has no `gh pr` command. Use `resolveReviewThread` GraphQL mutation via `gh-cli-api`:

```bash
gh api graphql -f query='
  mutation { resolveReviewThread(input: {threadId: "THREAD_ID"}) { thread { isResolved } } }'
```

## Scope Boundaries

Covers `gh pr review` only. Doesn't cover inline review comments (adding comments to specific code lines), requesting reviewers (see `gh-cli-prs-create`), or resolving review threads (see `gh-cli-api`).
