GH CLI PR Review — perform and dismiss PR reviews via `gh pr review`.

Approve:
```bash
gh pr review 123 --approve --body "LGTM"
```

Request changes — body required to explain what needs addressing:
```bash
gh pr review 123 --request-changes --body "Please address X before merging"
```

Comment-only (no verdict):
```bash
gh pr review 123 --comment --body "Thoughts inline"
```

Dismiss review — `--review-id` required; `gh` won't accept `--dismiss` without it:
```bash
gh pr review 123 --dismiss --review-id <review-id> --body "reason"
```
Obtain review ID:
```bash
gh pr view 123 --json reviews --jq '.reviews[].id'
```

Add reviewers — at creation use `gh pr create --reviewer`; after creation:
```bash
gh pr edit 123 --add-reviewer user
```
Covered by `gh-cli-prs-create`, not this skill.

Resolve review threads — no `gh pr` command; use GraphQL via `gh-cli-api`:
```bash
gh api graphql -f query='
  mutation { resolveReviewThread(input: {threadId: "THREAD_ID"}) { thread { isResolved } } }'
```

Scope: `gh pr review` only. Excludes inline review comments (specific code lines), requesting reviewers (→ `gh-cli-prs-create`), resolving threads (→ `gh-cli-api`).
