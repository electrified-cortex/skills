# GH CLI PRs

Entry point for pull request management via GitHub CLI. Handles common PR inspection commands and routes write operations to appropriate sub-skill.

## When to Use

Use when you need to inspect PRs (list, view, diff, check status) or want routing handled automatically for write operations. If you already know the sub-skill, dispatch it directly.

## Common Inspection Commands

List open PRs authored by you:

```bash
gh pr list --state open --author @me --json number,title,headRefName
```

View a PR with comments:

```bash
gh pr view 123 --comments
```

Show PR diff:

```bash
gh pr diff 123
```

Watch CI checks until complete:

```bash
gh pr checks 123 --watch
```

Check your own PR status summary:

```bash
gh pr status
```

## Sub-skills

| Sub-skill | Handles |
| --- | --- |
| gh-cli-prs-create/ | Open new PRs, convert drafts to ready |
| gh-cli-prs-review/ | Approve, request changes, dismiss reviews |
| gh-cli-prs-comments/ | Add, edit, delete PR comments |
| gh-cli-prs-merge/ | Merge, update branch, revert, close |

## Notes

- Use `--repo owner/name` when not inside a local clone of target repository.
- `gh pr checks --watch` is correct way to block until CI completes.
- Covers `gh pr` commands only. Git operations, branch protection rules, and CODEOWNERS configuration are out of scope.

## Scope Boundaries

Covers `gh pr` subcommands only. Doesn't touch git operations, branch protection, or CODEOWNERS.
